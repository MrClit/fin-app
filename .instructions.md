# Copilot Commit Message Instructions

When generating commit messages from source control changes:

- Always write the commit message in English.
- Use Conventional Commits format: <type>(<scope>): <short summary>.
- Allowed types: feat, fix, docs, style, refactor, test, chore.
- Use imperative mood (for example: "add", "fix", "remove").
- Keep the first line concise (around 50 characters when possible).
- Focus on what changed and why, not implementation minutiae.
- If needed, add a body separated by a blank line.
- In the body, wrap lines near 72 characters.
- If there is a linked issue, include: Closes #<number>.

Examples:

- feat(transactions): add date range filters
- fix(dashboard): correct negative balance calculation
- refactor(accounts): simplify monthly aggregation query
